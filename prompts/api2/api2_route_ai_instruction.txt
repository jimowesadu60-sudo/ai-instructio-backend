【本路线的核心分叉逻辑｜阶段 2.2 重做版】
当 `generation_route = "ai_instruction"` 时，三张方案卡片的差异，必须首先体现在：
“围绕同一主题，后续 AI 指令会采用怎样不同的讲法、组织重点与片段重心”。

注意：
这三张卡不是把同一条内容拆成：
- 开始
- 过程
- 结尾

也不是拆成：
- 起步
- 转折
- 收获

而必须拆成：
- 同一主题下，不同的讲法
- 同一主题下，不同的重点
- 同一主题下，不同的观看感受
- 同一主题下，不同的内容组织方式

用户看完三张卡后，应该感受到：
“这是同一条内容的三种不同讲法”，
而不是：
“这是同一条内容按时间顺序拆成三段”。

【AI 指令路线下唯一允许优先采用的分叉方式｜阶段 2.2 重做版】
在生成三张卡前，你必须先判断当前主题更接近哪一类，并按该类主题的“合法分叉维度”来分叉。

1. 如果主题偏个人经历类
优先从以下维度中选择 3 个最有差异的方向：
- 人物特质
- 感受重点
- 变化重点
- 收获重点

2. 如果主题偏方法经验类
优先从以下维度中选择 3 个方向：
- 痛点切口
- 方法组织方式
- 结果验证方式

3. 如果主题偏观点表达类
优先从以下维度中选择 3 个方向：
- 问题切口
- 冲突重点
- 判断落点

4. 如果主题偏日常记录 / vlog / 一天经历类
优先从以下维度中选择 3 个方向：
- 人设感
- 场景感
- 情绪感
- 节奏感
- 仪式感

说明：
- “人设感”指突出人物特质、生活方式、稳定气质、个人标签
- “场景感”指突出环境氛围、空间细节、沉浸状态
- “情绪感”指突出感受、内心状态、共鸣入口
- “节奏感”指突出行为流动、状态切换、片段推进
- “仪式感”指突出习惯、秩序、重复动作、生活审美

硬要求：
1. 不允许默认按时间顺序切三张卡
2. 不允许默认按“体验 / 转折 / 收获”切三张卡
3. 三张卡必须是“讲法不同”，不是“阶段不同”
4. 如果输出读起来像“起步—变化—总结”，则当前结果不合格

【card_title 的唯一任务｜阶段 2.2 重做版】
card_title 必须回答：
“这张卡准备怎么讲当前主题？”

它不是：
- 方法说明
- 生成说明
- 分析标题
- 策划语言
- 提示词语言

card_title 必须满足：
1. 必须像前端真实可选方案标题
2. 必须短、顺、自然
3. 必须让用户一眼读懂
4. 必须直接围绕当前主题
5. 必须体现这张卡最核心的讲法差异
6. 标题优先控制在 8～16 个字
7. 日常记录 / 经历类主题优先使用自然中文表达，必要时可用第一人称
8. 不要写成“从……切入”“关注……变化”“总结……”这类分析腔
9. 不要出现“版 / 路线 / 组织 / 生成 / 表达风格 / 完整版 / 基础版”这类系统感强的词
10. 不要堆叠抽象词，例如“成长变化启示”“整体体验价值”

更符合要求的标题方向示意：
- 我为什么总爱在咖啡厅学习
- 原来办公室里最让人累的是这种氛围
- 这家榴莲自助到底值不值得推荐
- 我是怎么慢慢适应国外生活的
- 真正让我坚持下来的，其实是这个习惯

不符合要求的标题方向示意：
- 从学习环境谈成长变化
- 分享我的学习仪式与体验
- 描述我在咖啡厅的学习感受
- 关注学习过程中的挑战与变化
- 总结学习收获与收获

注意：
以下示意只用于说明“标题自然度”和“讲法差异”的写法原则，
不是固定句库，也不得复用其中的具体场景、动作、主题名词。
你必须根据当前主题类型、主目标和条件语义，重新生成属于该主题的标题。

【card_summary 的唯一任务｜阶段 2.2 重做版】
card_summary 必须回答：
“这一版看起来会是什么感觉？它会把哪一面内容放大出来？”

它不是：
- 对标题的复述
- 系统对讲法的分析
- 写作步骤说明
- “先……再……最后……”的流程说明

card_summary 必须满足：
1. 要让用户一眼看懂这一版会重点呈现什么
2. 要告诉用户这一版更会放大主题中的哪一面
3. 要让用户感受到这版和另外两版的差异
4. 必要时要自然回应主目标，但不能写成“因为当前输入……”
5. 要像前端卡片摘要，不像系统解释
6. 尽量使用自然中文，避免抽象词和分析词
7. 不要复述标题里的核心词再说一遍
8. 不能写成“这一版会围绕……组织内容”
9. 不能写成“适合想表达……的用户”
10. 不能写成“先……再……最后……”

更符合要求的摘要方向示意：
- 先把最容易被人看见的状态抛出来，再顺着带出你为什么会慢慢形成这种习惯。
- 重点会落在那些最容易被忽视、但最能说明问题的细节上，让主题更有代入感。
- 会先把最容易引发情绪反应的那一面讲出来，再自然带到你的真实判断里。
- 更像在看一个真实片段，环境、人物反应和内容重点会一起把人带进去。
- 会先把“值不值得”“为什么会这样”这类最想知道的问题抛出来，再往下展开。

不符合要求的摘要方向示意：
- 本版通过讲述我在咖啡厅的学习环境来探讨个人成长。
- 这个版本将从我的第一次学习体验讲起，逐渐带出对学习的强大感受。
- 这一版会围绕主题组织内容，并给出更清楚的展开路径。

硬要求：
1. 三张卡的 card_summary 不允许近似重复
2. 三张卡的 card_summary 不允许只换几个词
3. summary 必须比 title 多提供一层“观看感受/呈现重点”的信息

补充说明：
以上示意只用于说明“摘要该如何提供观看感受和呈现重点”，
不得复用其中的具体主题名词、场景名词或动作名词。

【最终要求】
当 route 为 `ai_instruction` 时，你输出的 3 个方案必须做到：
- 用户一眼能看懂差异
- 三版后续生成方式确实会不同
- API3 能据此展开成不同的专业脚本母数据与分段 AI 指令
- 平台适配真实体现在执行路线差异中
- 结果既能服务前端卡片，也能服务后续 AI 指令生成

-----------------------------------
【最终自检｜生成前必须检查】
在输出三张卡片前，你必须自检：

1. 只看 card_title，用户能不能看出当前主题是什么？
2. 只看 card_title，三张卡是不是像“同一主题的不同讲法”？
3. 三张标题里，有没有出现“基础版 / 完整版 / 快速版 / 组织版”这类空泛模板命名？
4. 如果把 topic 换掉，当前标题是否仍然成立？
- 如果仍然成立，说明标题太泛，不合格，必须重写
5. 只看标题，普通用户能否立刻读懂？
6. 标题读起来是否像自然中文，而不是系统策略说明？
7. 如果把“版 / 路线 / 组织 / 生成 / 风格”这些词去掉后标题会更自然，说明当前标题不合格，必须重写
8. 标题是否适用于“前端卡片直接展示”，而不是更像内部 prompt 字段名？
9. 只看 highlight_tags，用户能否快速判断这张卡在讲什么？
10. 如果标签更像“生成方式说明”而不是“内容重点提示”，则当前输出不合格，必须重写。

【最终自检｜Step 2 新增】
在输出三张卡片前，你必须自检：

1. 三张卡的差异是不是建立在“讲法不同”上？
2. 三张卡是不是仍然主要靠“阶段先后”来区分？
3. 如果把“开始 / 转折 / 收获”这些词删掉，三张卡还成立吗？
- 如果不成立，说明仍然只是阶段切分，不合格
4. 用户是否能感觉到：
- 这三张卡是在提供三种不同表达方案
- 而不是把同一条内容拆成三个阶段
5. 如果换掉本次平台 / 目标 / 时长 / 表达方式，这三张卡还完全成立吗？
- 如果完全成立，说明它们太泛，不合格
6. 用户看完三张卡，能否感觉到：
- 这三张卡是围绕当前主题的不同讲法
- 同时又是基于本次输入条件做出的不同选择
7. 每张卡的 core_structure_path 是否至少有 3 项？
8. 每一项是否都是非空、可执行、可继续展开的内容步骤？
9. 如果删掉标题，只看 core_structure_path，是否也能看出这张卡准备怎么展开？
10. 若任意一张卡的 core_structure_path 少于 3 项，则整份输出不合格，必须重写。
11. 每个 highlight_tags 是否都短、直白、像内容特征，而不是步骤说明？
12. card_summary 是否还带有明显“整理腔 / 分析腔 / AI 味”？
13. 这三张卡是否抓住了用户输入里的情绪和表达意图，而不是只抓住了主题名词？
14. 用户看完三张卡，是否能感觉到：
- 这三张是在回应我这次真正想说的话
- 而不是只围绕主题表面在生成三种方案
15. “适合”后面的句子是否还像系统解释，而不像自然说明？
16. card_summary 是否还带有“因为此时输入……”“更适合当前条件……”这类 AI 判断腔？
17. card_title 是否更像真实卡片标题，而不是“讲法说明句”？
