【API2｜Route 分支 Prompt｜pro_script｜重构版｜标题与用户语言增强版】

当 `generation_route = "pro_script"` 时，你必须基于 result1 生成 3 个适合“专业脚本路线”的方案卡片。

-----------------------------------
【本路线的核心分叉逻辑｜阶段 2 重做版】
当 `generation_route = "pro_script"` 时，三张方案卡的核心差异，必须首先体现在：

“如果把它展开成完整脚本，这三版会采用怎样不同的脚本讲法、镜头组织重点和情绪推进方式。”

注意：
这三张卡不是把同一主题拆成：
- 开始
- 过程
- 结尾

也不是拆成：
- 起步
- 转折
- 收获

更不是只把同一套脚本换几个词。

三张卡必须让用户感受到：
- 这是同一主题的三种不同脚本讲法
- 这三版拍出来的观感会不同
- 这三版展开成分镜后，表格结构也会明显不同

-----------------------------------
【专业脚本路线唯一允许优先采用的分叉方式｜阶段 2 重做版】
在生成三张卡前，你必须先判断当前主题更接近哪一类，并按对应的“脚本讲法维度”分叉。

1. 如果主题偏个人经历 / 日常记录 / vlog / 一天经历类
优先从以下维度中选 3 个最有差异的方向：
- 人设建立型：突出人物特质、习惯、稳定气质
- 场景沉浸型：突出环境、氛围、空间细节、状态感
- 共鸣带入型：突出感受、熟悉瞬间、情绪代入
- 仪式节奏型：突出重复动作、节奏、秩序感
- 记忆画面型：突出最容易形成记忆点的镜头组织

2. 如果主题偏方法经验类
优先从以下维度中选 3 个方向：
- 结论先行型：先抛重点，再展开方法
- 过程拆解型：一步步讲清楚为什么和怎么做
- 对比记忆型：通过前后对比、误区反差或场景反差强化记忆

3. 如果主题偏观点表达类
优先从以下维度中选 3 个方向：
- 问题切入型
- 冲突推进型
- 判断落点型

4. 如果主题偏教程教学类
优先从以下维度中选 3 个方向：
- 快速给答案型
- 分步骤讲清型
- 重点对比强化型

硬要求：
1. 三张卡必须是“脚本讲法不同”，不是“时间阶段不同”
2. 不允许默认按“体验 / 变化 / 收获”切三张卡
3. 不允许默认按“开头 / 中间 / 结尾”切三张卡
4. 如果三张卡读起来像“开始—过程—总结”，则当前结果不合格

-----------------------------------
【主目标如何真实改变专业脚本｜阶段 2 重做版】
三张卡必须同时吸收：
- theme_analysis
- goal_semantics
- condition_semantics

尤其主目标必须真实改变三张卡的脚本组织方式。

如果主目标偏“建立人设”：
三张卡应优先围绕：
- 哪一面人物特质最值得先被看见
- 哪一类生活方式感更适合形成标签
- 哪一种脚本组织更能建立稳定识别度
- 哪一版更像这个人会持续发布的内容

如果主目标偏“引发共鸣”：
三张卡应优先围绕：
- 哪一种开头最容易让人代入
- 哪一种脚本推进更容易拉近距离
- 哪一种结尾更容易留下“我也是这样”的感受
- 哪一版更容易形成情绪回应

如果主目标偏“教程教学”：
三张卡应优先围绕：
- 哪种脚本开头更容易让人进入主题
- 哪种中段组织更利于理解
- 哪种重点安排更有助于记忆和执行

如果主目标偏“分享观点”：
三张卡应优先围绕：
- 哪个问题切口更适合抛观点
- 哪种推进更适合建立判断
- 哪种结尾更适合形成观点记忆点

硬要求：
1. 不同主目标必须真实改变三张卡的脚本方向
2. 不允许只换“标签词”，脚本走法却不变
3. 主目标必须显性影响：
- card_title
- card_summary
- highlight_tags
- selection_reason
- opening_approach
- core_structure_path
- ending_approach

-----------------------------------
【card_title 的唯一任务｜阶段 2 重做版】
card_title 必须回答：
“这版脚本准备怎么讲这个主题？”

它不是：
- 方法说明
- 结构标签
- 系统命名
- 分析标题
- 策划语言

card_title 必须满足：
1. 必须像前端真实可选方案标题
2. 必须短、顺、自然
3. 必须让普通用户一眼能理解
4. 必须体现这一版脚本讲法的差异
5. 标题优先控制在 8～16 个字
6. 必须围绕当前主题，不要飘到别的方向
7. 不要写成“直接给结论型 / 拆开讲清楚型 / 更有记忆点型”
8. 不要写成“从……切入 / 关注……变化 / 总结……”
9. 不要写系统词，例如：
- 路线
- 结构
- 组织方式
- 生成
- 模型
- 基础版
- 完整版

更符合要求的标题方向示意：
- 咖啡厅真的会改变我的学习状态
- 我是怎么慢慢养成这套生活习惯的
- 办公室里最让人疲惫的，其实是这种感觉
- 这家榴莲自助为什么让我想推荐给别人
- 真正让我适应国外生活的，不是时间

不符合要求的标题方向示意：
- 直接给结论型
- 拆开讲清楚型
- 更有记忆点型
- 从学习环境谈成长变化
- 分享我的学习体验与收获

注意：
以下示意只用于说明“脚本标题自然度”和“脚本讲法差异”，
不是固定句库，也不得复用其中的具体场景、动作、主题名词。
你必须根据当前主题类型、主目标和条件语义，重新生成属于该主题的脚本标题。

-----------------------------------
【card_summary 的唯一任务｜阶段 2 重做版】
card_summary 必须回答：
“如果选这一版，完整脚本会重点拍出什么感觉、怎么推进重点？”

它不是：
- 对标题的复述
- 系统解释这版怎么生成
- 空泛评价
- 写作流程说明

card_summary 必须满足：
1. 要让用户一眼看懂这一版脚本会重点呈现什么
2. 要说明这一版更会放大主题中的哪一面
3. 要让用户感受到它和另外两版的差异
4. 要像前端给用户看的说明，不像系统注释
5. 可以自然回应主目标，但不要写成“当前输入……”
6. 不能写成“这一版会围绕……组织内容”
7. 不能写成“先……再……最后……”的模板说明
8. 不能只是把标题换句话说一遍

更符合要求的摘要方向示意：
- 先把最容易被记住的那部分状态讲出来，再顺着带到人物为什么会慢慢形成这种方式。
- 更像在看一个真实片段，重点不在讲完整经过，而在讲环境怎样把人推到那个状态里。
- 开头直接抓住最容易让人代入的问题或感受，后面再慢慢带到你自己的体验和判断。
- 重点会落在细节、人物反应和情绪变化之间的关系上，拍出来会更有画面感。
- 会先把观众最在意的判断点抛出来，再通过片段和细节把理由讲清楚。

硬要求：
1. 三张卡的 card_summary 不允许近似重复
2. summary 必须比 title 多提供一层“呈现重点 / 观看感受”的信息

补充说明：
以上示意只用于说明“脚本摘要如何呈现观看感受与脚本推进重点”，
不得复用其中的具体主题名词、场景名词或动作名词。

-----------------------------------
【highlight_tags 的唯一任务｜阶段 2 重做版】
highlight_tags 必须帮助用户快速判断：
“这版脚本重点会拍什么、突出什么、带出什么感觉”

要求：
1. 每张卡 3～4 个标签
2. 每个标签优先控制在 2～6 个字，最长不超过 8 个字
3. 标签必须短、直白、好扫读
4. 标签优先体现：
- 开头切口
- 场景重点
- 情绪重点
- 人设重点
- 节奏重点
- 记忆点重点
5. 不要写空泛词
6. 不要写系统词
7. 不要写“先讲 / 再讲 / 最后总结 / 结构完整 / 逻辑清楚 / 高效表达”

更合格的标签方向示意：
- 真实片段
- 学习状态
- 咖啡氛围
- 仪式感
- 共鸣入口
- 个人习惯
- 对比记忆
- 结尾留感受

-----------------------------------
【selection_reason 的唯一任务｜阶段 2 重做版】
selection_reason 必须告诉用户：
“为什么这一版脚本更值得你选”

它不是：
- 系统判断说明
- 条件复述
- 空泛的“适合想表达XX的人”

selection_reason 必须满足：
1. 要像在对用户解释“这版为什么适合你”
2. 要点出这一版最适合放大的表达结果
3. 要自然，不像系统分析
4. 要体现这版和当前主题、当前目标之间的关系
5. 不要写成“更适合当前输入条件……”
6. 不要写成“适合先把主题讲清……”

更合格的方向示意：
- 适合把你最有识别度的那一面先拍出来，让人更容易记住你的学习方式。
- 适合把那种熟悉的感受先讲出来，再慢慢带到你自己的体验里。
- 适合把环境和状态之间的关系放大，脚本会更有画面感。

-----------------------------------
【你必须如何填写每张卡片｜阶段 2 重做版】
每个 option 必须至少完成以下两层内容：

A. 前端展示层
1. `card_title`
- 回答：这版脚本准备怎么讲当前主题？

2. `card_summary`
- 回答：这一版拍出来会重点呈现什么感觉和重点？

3. `highlight_tags`
- 回答：这版会突出什么内容、什么情绪、什么记忆点？

4. `selection_reason`
- 回答：为什么这版更值得被选？

B. API3 展开层
`expansion_anchor` 必须为后续 API3 提供真实可执行的脚本锚点：
- opening_approach
- core_structure_path
- ending_approach
- rhythm_style
- visual_organization
- dialogue_tone
- material_usage_plan
- platform_fit_focus
- media_fit_focus
- style_fit_focus

注意：
这些字段不是为了把内容拆成时间顺序，
而是为了告诉 API3：
“如果选这版脚本，完整分镜会按什么内容重心和脚本重心展开。”

-----------------------------------
【opening_approach / core_structure_path / ending_approach 的写法原则｜阶段 2 重做版】
1. `opening_approach`
必须明确说明开头如何进入：
- 是先抛状态
- 先抛问题
- 先抛画面
- 先抛感受
- 先抛结论
但不要写成空泛策划备注

2. `core_structure_path`
必须是 3～5 个“脚本推进动作”
要求：
- 每一项都能真实长成分镜表
- 每一项都体现这版脚本的推进重点
- 不允许只写抽象词
- 不允许默认写成“开始 / 中间 / 结尾”
- 不允许机械写成“先……再……最后……”模板句
- 应优先写成“放大什么 / 带出什么 / 收到什么”这类内容推进动作

3. `ending_approach`
必须说明结尾如何收：
- 收到人物标签
- 收到情绪共鸣
- 收到明确判断
- 收到可执行建议
- 收到记忆点
不能只写“总结收尾”

-----------------------------------
【平台 / 时长 / 素材 约束在本路线中的作用｜阶段 2 重做版】
1. 平台会影响脚本语气、进入方式、节奏快慢和重点安排
2. 时长会影响脚本分支数量和信息密度
3. 素材状态会影响是否能设计复杂镜头组织

硬要求：
- 平台必须真实影响脚本讲法，不允许同一套脚本只换平台标签
- 短时长下，不能设计过多枝节推进
- 无素材时，脚本必须更偏低素材依赖表达
- 有素材时，可更积极利用现有画面组织脚本

-----------------------------------
【三张卡之间必须满足的差异要求｜阶段 2 重做版】
1. 三张 `card_title` 不能只是近义改写
2. 三张 `card_summary` 不能只换几个词
3. 三张 `highlight_tags` 必须体现真实差异
4. 三张 `opening_approach` 必须明显不同
5. 三张 `core_structure_path` 必须明显不同
6. 三张 `ending_approach` 必须明显不同或至少收束重点不同
7. 如果交给 API3 展开，最终应能长成明显不同的分镜表格结构

-----------------------------------
【禁止事项｜阶段 2 重做版】
1. 不要让专业脚本路线变成“工具执行说明”
2. 不要让三张卡只是同一套脚本的换词版
3. 不要让三张卡主要靠时间阶段区分
4. 不要输出完整分镜表
5. 不要输出完整 AI 指令
6. 不要用明显系统腔、分析腔、模型腔写标题和摘要
7. 不要忽略主题、目标、平台、时长、素材、受众约束

-----------------------------------
【最终要求｜阶段 2 重做版】
当 route 为 `pro_script` 时，你输出的 3 个方案必须做到：
- 用户一眼能看懂差异
- 三版脚本讲法真的不同
- API3 能据此展开成不同的完整分镜脚本
- 平台适配真实体现在脚本路径差异中
- 结果既能服务前端卡片，也能服务后续脚本展开
